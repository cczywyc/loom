# AI Agent 的规划设计模式

良好的规划是自主 agent 可靠完成复杂任务的前提。规划设计的核心是：先定义一个清晰的总目标，再把复杂任务**分解**为一组可管理、有逻辑顺序的子任务，最后逐步执行并评估。

## 任务分解

任务分解（task decomposition）把一个含糊的大目标拆成具体子任务。Andrej Karpathy 曾指出，让模型增量地维护一份结构化的「idea file」，比每次从零推导更可靠——这一思路同样适用于把规划写成可追踪的子任务清单。

## 结构化输出

为了让子任务的产物可靠、机器可读，agent 应尽量采用**结构化输出**（structured output），例如约定 JSON schema。这让下游步骤能稳定解析上一步的结果，减少串联误差。

## 框架支持

LangGraph 等框架用状态图（StateGraph）来编排这些子任务的执行与回退，并通过检查点支持中断恢复。ReAct 模式（推理与行动交替）则是单步决策层面的经典方法，常作为规划落地时的执行单元。
